### 参考基准格式（用户指定）

`docs/tutorials/index.md` 表格格式：
```markdown
| 分类 | 教程 | 一句话介绍 |
|------|------|------------|
| AI 编程 | [Claude Code 使用指南](/tutorials/claude-code) | 已完成的教程 |
| 开发环境 | （待补）VS Code 效率插件推荐 | 待补教程，无超链接 |
```

关键点：**已写的保留真实链接；未写的加"（待补）"前缀并不加链接**，对读者诚实。

## 四、潜在依赖与考虑

| 项目 | 说明 |
|---|---|
| "已完整写 / 草稿 / 完全空"的判定标准 | 完整写的：`docs/ml/svm.md`、`docs/dl/transformer-1.md`、`docs/tutorials/claude-code.md`、`docs/papers/transformer-paper.md`、`docs/math/linear-algebra.md` —— 共 **5 篇完整 + 1 个首页 + 1 个关于 + 1 个 release**；其余 50+ 为骨架（只有标题没有正文）。判定会在执行阶段用 Read 逐文件抽查前 30 行是否有正文内容来确认。 |
| 首页章节目录是否完全删除？ | 用户没要求删除目录，只说"改关于本站描述、删更新、其他模块待补"。推荐**保留目录 + 待补徽章**（逼迫自己按计划写），不建议完全删除。计划里默认方案 A，用户可在审批时要求换方案 B。 |
| 侧边栏 sidebar 配置要不要去掉？ | 用户没提。当前 `config.ts` 侧边栏写了 60+ 条链接，指向的都是空骨架但能打开（不会 404）。**保持**，因为修改计划不含侧边栏；若用户后续想隐藏空章节，可再单独提。 |
| 构建风险 | 修改后必须 `npm run docs:build` 验证：1) 有没有 dead link（完全去掉链接的条目不应被链接到）；2) 表格语法正确；3) 新增 CSS 类选择器无误 |

## 五、风险处理

| 风险 | 处理方案 |
|---|---|
| 误把"有完整内容"的 5 篇改成「待补」 | 执行 Step 3-6 前，对每个提到的章节先用 Read 读取前 50 行，如果有 ≥ 300 字正文（非仅标题和 TOC）则标记为"已完成"，保持链接不改动 |
| 表格格式不渲染 / Markdown 表格少列错位 | 每个模块改完立刻 `docs:build` 验证；或用 VitePress 开发模式 `docs:dev` 实时预览 |
| 首页卡片待补样式影响移动端 | custom.css 新增类全部加 `@media (max-width: 768px)` 适配，或干脆不在卡片右上角加徽章（改成在标题前加文字前缀，最稳妥） |
| 推送后 Pages 报错 | 先本地 `docs:build` 全绿再 push（此项目已知 push 即 Actions 自动部署） |
